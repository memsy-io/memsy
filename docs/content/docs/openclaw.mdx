Add Memsy to your [OpenClaw](https://openclaw.ai) gateway to give your agent persistent memory across every connected channel. Store a decision from Telegram, recall it from Slack — the memory is channel-agnostic.

## Requirements

- Node.js 18+
- [OpenClaw](https://openclaw.ai) gateway running
- Memsy API key — [app.memsy.io](https://app.memsy.io) → Settings → API Keys

## Install

<Steps>
  <Step title="Clone and run the installer">
    ```bash
    git clone https://github.com/memsy-io/memsy
    cd memsy/plugins/openclaw
    ./install.sh
    ```

    The installer merges `mcpServers.memsy` into `~/.openclaw/openclaw.json` and copies two skills to `~/.openclaw/workspace/skills/`.
  </Step>

  <Step title="Set your API key">
    The recommended way persists the key across restarts:

    ```bash
    openclaw config set env.MEMSY_API_KEY "msy_..."
    ```

    Alternatives:

    ```bash
    # .env file (survives daemon restarts)
    echo "MEMSY_API_KEY=msy_..." >> ~/.openclaw/.env

    # Session only
    export MEMSY_API_KEY=msy_...
    ```

    <Info>
      `openclaw config set env.MEMSY_API_KEY` writes the key into `~/.openclaw/openclaw.json` and injects it at every gateway start. Use `~/.openclaw/.env` instead if you run OpenClaw as a systemd/LaunchAgent service.
    </Info>
  </Step>

  <Step title="Verify">
    In any connected chat app, send:

    > *"What do we know about our Postgres decision?"*

    Your OpenClaw agent should trigger the `memsy-recall` skill and call `memsy_search`.
  </Step>
</Steps>

## Manual config

Add to `~/.openclaw/openclaw.json`:

```json
{
  "mcpServers": {
    "memsy": {
      "command": "npx",
      "args": ["-y", "@memsy-io/mcp"],
      "env": {
        "MEMSY_API_KEY": "msy_..."
      }
    }
  }
}
```

Restart OpenClaw after editing the config.

## Skills

Two skills install to `~/.openclaw/workspace/skills/`:

| Skill | Trigger phrases |
|---|---|
| `memsy-recall` | "what did we decide", "remember when", "context on X", "do we have anything about Y" |
| `memsy-remember` | "remember that", "save this decision", "note that", "for future reference" |

Skills are available to your agent alongside MCP tools across every connected channel. Memory stored from one channel is searchable from any other.

## Multi-org

Configure profiles in `~/.memsy/config.json` (see [MCP docs](/docs/mcp#multi-org)), then ask your agent:

- *"What orgs do I have?"* → calls `memsy_list_orgs`
- *"Switch to my work org"* → calls `memsy_use_org`

| Capability | Supported |

## Troubleshooting

### MCP shows "disconnected"

Run the server directly to see the error:

```bash
MEMSY_API_KEY=msy_... npx -y @memsy-io/mcp
```

### Skills not triggering

Confirm `~/.openclaw/workspace/skills/memsy-recall/SKILL.md` exists. Re-run `./install.sh`.

### Wrong memories returned

Ask your agent: *"What Memsy orgs do I have?"* to check the active profile. Switch with *"Use my work Memsy org."*
