**Memsy** turns the raw events your AI app produces — user messages, assistant replies, tool results — into structured, searchable long-term memory.

The `memsy` package is the official Python client for the Memsy API: a thin, typed HTTP client with built-in retries, usage tracking, and a sync/async twin.

```bash
pip install memsy
```

## What the SDK does

- Ships events to Memsy with a single `ingest()` call (batched, async on the server).
- Retrieves semantically relevant memories with `search()`.
- Surfaces usage, rate-limit, and plan information on every response.
- Maps HTTP errors to strongly-typed exceptions (`AuthenticationError`, `RateLimitExceeded`, `UsageLimitExceeded`, …).
- Retries rate-limited requests automatically with exponential backoff.

## What it does not do

The SDK is a thin HTTP client. It does not run memory extraction, embeddings, or vector search locally — all processing happens on the Memsy platform.
